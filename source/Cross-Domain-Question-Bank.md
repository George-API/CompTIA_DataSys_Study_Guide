Cross-Domain Practice Question Bank

Why this, and only this

The 5 domain files optimize for encoding (understanding and organizing information). What's missing is a resource that optimizes for retrieval — the act of pulling information from memory under exam-like conditions. Decades of cognitive science research (testing effect, desirable difficulty, interleaving) show that:





Active recall (answering a question) produces 2-3x better retention than re-reading the same material



Interleaving (mixing topics) forces the brain to discriminate between similar concepts, which is exactly what MCQ distractors exploit



Elaborative feedback (explaining why the right answer is right and wrong answers are wrong) closes knowledge gaps in real time

A single well-built question bank achieves all three.

File





00-Practice-Questions.md — placed first alphabetically so it's the natural starting point for study sessions

Structure

Organization: by question type, not by domain

Questions grouped by CompTIA verb pattern, with domains interleaved within each group. This prevents the brain from settling into "Domain 3 mode" and forces genuine retrieval.

## Compare and Contrast
(questions pulling from Domains 1, 2, 3, 4 — interleaved)

## Given a Scenario
(questions pulling from Domains 1, 2, 3, 4 — interleaved)

## Explain / Identify
(questions pulling from all 5 domains — interleaved)

## Describe / Recognize
(questions pulling from Domains 4, 5 — interleaved)

Question format

Each question follows this pattern:

**Q12.** A DBA discovers that an ORM is generating SELECT * queries against
a 40-column table when the application only uses 3 columns. CPU and I/O
on the database server have increased 30% since the ORM update.
What should the DBA do FIRST?

A. Add more RAM to the database server
B. Review the SQL code generated by the ORM
C. Rewrite the application to use stored procedures
D. Create a covering index on the 3 columns

<details><summary>Answer</summary>

**B.** The exam-critical sequence for ORM impact (1.3) starts with reviewing
the generated SQL before taking any other action. A and D address symptoms,
not root cause. C may be a valid long-term solution but is not the first step.

**Domain:** 1.0 | **Objective:** 1.3 | **Verb:** Explain

</details>

Key design choices:





Collapsible answers (<details> tags) — prevents accidental exposure; forces a genuine retrieval attempt before checking



Domain/objective tag on every answer — links back to the study guide section for targeted review of weak areas



Wrong-answer explanations — explains why each distractor is wrong, which is where most exam points are actually lost



"FIRST" / "BEST" / "MOST" qualifiers — mirrors how CompTIA frames scenario questions to test priority, not just knowledge

Coverage targets





~8-10 questions per domain, weighted by exam percentage (so ~12 for Domain 3, ~6 for Domain 5)



~40-45 questions total (enough to be useful, not so many that the file becomes a wall)



At least one question per objective (1.1, 1.2, ... 5.2) to ensure full coverage



Mix of difficulty: some test straight recall, some require applying concepts to a novel scenario

Topics that get extra attention (high-confusion areas)

These are concepts the exam is known to test through close distractors:





DDL vs. DML vs. DCL vs. TCL (which sublanguage does X belong to?)



Differential vs. incremental backup (restore chain differences)



RPO vs. RTO (which drives backup frequency vs. architecture?)



View vs. materialized view (when to use which)



Static vs. dynamic masking



Logical vs. physical vs. view schema



Replication vs. mirroring vs. log shipping (RPO/RTO trade-offs)



Trigger vs. stored procedure (automatic vs. called)



PII vs. PHI vs. PCI DSS (which regulation/standard applies?)

What this does NOT include (and why)





Flashcards file — flashcards are effective but are better served by Anki/Quizlet (tool-native spaced repetition). A static markdown file can't do SRS properly, so it would just be a worse version of the question bank.



Concept map / mind map — useful for visual learners but hard to maintain in markdown and adds little over the already well-structured domain files.



Acronym glossary — Exam-Objectives.md already has the official acronym list (lines 373-426), and every acronym is expanded inline in the study guides. Duplicating it adds no value.
